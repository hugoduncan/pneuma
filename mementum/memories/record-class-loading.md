pneuma.lean.blueprint uses `:import` to load record classes (CapabilitySet, MealyHandlerSet, etc.) for multimethod dispatch. These classes only exist after the namespace defining the defrecord has been loaded. Any test or namespace that requires pneuma.lean.core (which transitively requires blueprint) must first require all formalism and morphism namespaces that define records. This is a load-time error — skip-meta and kaocha filtering cannot prevent it because the NS is loaded before tests are filtered. Solution: either add explicit requires in the test ns, or separate such tests into a different source path (test-regression-lean/) so kaocha never scans them.
