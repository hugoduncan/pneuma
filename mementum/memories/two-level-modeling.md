🔁 pneuma has two structural domain models at different levels: the domain model (doc/structural-domain-model.md) describes WHAT pneuma checks (formalisms, projections, morphisms, gaps). The architecture model (doc/option-a-formalism.md) describes HOW pneuma is built (namespaces, protocols, records, registries). Appendix D of the architecture model maps between them. This two-level pattern — domain model vs implementation model with explicit mapping — was valuable for separating concerns and could be reused in other projects.
