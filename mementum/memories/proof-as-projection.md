🎯 Lean proofs are a projection of Pneuma's gap report, not manually written. The system emitter (pneuma.lean.system/emit-system-lean) runs gap-report internally and emits `decide` for conforming morphisms, `sorry` with diagnostics for failing ones. This means: conforming specs compile in Lean (proof verified by kernel), failing specs break the Lean build (sorry is a compile error in CI). The proof is the gap report in a different notation — same data, different checker.
