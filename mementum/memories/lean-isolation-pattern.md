🎯 Optional protocol layers use extend-protocol in a sibling namespace layer, not additional methods on core protocols. The lean layer (pneuma.lean.*) demonstrates: define new protocols in pneuma.lean.protocol, extend existing records in per-formalism namespaces, require all extensions in pneuma.lean.core. The domain model can describe ->lean as a fifth projection functor; the architecture realizes it via separate protocols. This pattern applies to any optional projection target (e.g., Alloy, TLA+).
