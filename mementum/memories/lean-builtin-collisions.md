Type schema names emitted to Lean must avoid Lean 4 builtin names. `Void`, `Bool`, `Nat`, `String`, `List`, `Option`, `Unit` etc. are already defined in Lean's prelude. Opaque declaration emission already skips these (via a known-builtins set), but user-defined type names in TypeSchema formalisms are not filtered — they emit as `opaque Void : Type := Unit` which conflicts. Fix: choose non-colliding names (e.g., `UnitResult` instead of `Void`).
