Kaocha ns-patterns like `.*-test$` match broadly — `lean-compile-test` matches `.*-test$`. Metadata-based filtering (skip-meta) does not prevent namespace loading, only test execution. To truly isolate test namespaces that have incompatible load requirements (e.g., lean tests needing record classes), use separate test directories with separate kaocha suites. Current layout: test/ (unit), test-regression/ (model-based, fast), test-regression-lean/ (lean compilation, slow). Each has its own kaocha suite ID.
